GET Health Check

GET /health

Check if the API server is running and get system information.

Response Example:

{

  "timestamp": "2024-01-15T10:30:00.000Z",
  "uptime": 3600,
  "version": "1.0.0"

Download APIs

Endpoints for file downloads, URL validation, and batch processing.

POST File Download Info

POST /api/download/file

Get information about a downloadable file from a URL.

Request Parameters:

Parameter Type Required Description
url string Yes Valid URL of the file to analyze

Request Example:

curl -X POST https://api-hub.onrender.com/api/download/file \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com/file.pdf"}'

Response Example:

{
  "success": true,
  "url": "https://example.com/file.pdf",
  "fileInfo": {
    "size": "1048576",
    "type": "application/pdf",
    "downloadable": true
  },
  "downloadUrl": "https://example.com/file.pdf"
}

POST Batch URL Processing

POST /api/download/batch

Process multiple URLs simultaneously (1-10 URLs).

Request Parameters:

Parameter Type Required Description
urls array Yes Array of URLs to process (max 10)

AI APIs

Artificial Intelligence powered text processing endpoints.

POST Text Summarization

POST /api/ai/summarize

Generate a summary of the provided text.

Request Parameters:

Parameter Type Required Description
text string Yes Text to summarize
length string No Summary length: short, medium, long (default: medium)

Request Example:

curl -X POST https://api-hub.onrender.com/api/ai/summarize \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Your long text here...",
    "length": "medium"
  }'

Request Parameters:

Parameter Type Required Description

POST Text Generation

POST /api/ai/generate

Generate text based on a prompt.

Request Parameters:

Parameter Type Required Description
prompt string Yes Prompt for text generation
maxLength integer No Maximum length of generated text (50-500)

Image APIs

Image processing and manipulation endpoints.

POST Image Resize

POST /api/image/resize

Resize an image to specified dimensions.

Request Parameters:

Parameter Type Required Description
image file Yes Image file to resize
width integer No Target width (1-4000, default: 300)
height integer No Target height (1-4000, default: 300)

Request Example:

curl -X POST https://api-hub.onrender.com/api/image/resize \
  -F "image=@your-image.jpg" \
  -F "width=800" \
  -F "height=600"

POST Image Filter

POST /api/image/filter

Apply filters to an image.

Available Filters:

  • grayscale - Convert to black and white
  • blur - Apply blur effect
  • sharpen - Enhance image sharpness
  • brightness - Increase brightness
  • contrast - Adjust contrast
  • sepia - Apply sepia tone

Error Handling

APIs return standard HTTP status codes and JSON error responses.

Common Status Codes:

Code Meaning
200 Success
400 Bad Request - Invalid parameters
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error

Error Response Format:

{
  "error": "Validation Error",
  "message": "Invalid URL format provided",
  "details": {
    "field": "url",
    "value": "invalid-url"
  }
}
